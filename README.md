# Data Infrastructure & Platform for Data Science

This project was developed as part of a university course "Infrastruktur dan Platform untuk Sains Data" (Semester 3). The focus of this project is to understand how data is processed, managed, and prepared within a data science workflow using a structured and scalable approach.

## Project Overview
The objective of this project is to build a structured data processing workflow that transforms raw data into a clean and analyzable format. This includes data ingestion, preprocessing, transformation, and preparation for analysis.

## Dataset Description
The dataset used in this project consists of transactional and/or structured data (CSV format), containing:

- Multiple features with numerical and categorical variables
- Raw data requiring cleaning and preprocessing
- Missing values and inconsistencies for real-world simulation

Due to file size limitations, additional datasets (Tugas_IFSD_3) are not included in this repository.

## Key Processes

### 1. Data Ingestion
- Loading data from CSV files
- Initial data inspection

### 2. Data Cleaning
- Handling missing values
- Removing duplicates
- Data type correction

### 3. Data Transformation
- Feature selection
- Feature engineering
- Data normalization or standardization

### 4. Data Preparation
- Structuring dataset for analysis or modeling
- Ensuring data consistency and quality

## Tools & Technologies
- Python
- Pandas

## Key Learnings
- Understanding the importance of data preprocessing in data science workflows
- Building structured and reproducible data pipelines
- Handling real-world messy datasets
- Preparing data for downstream analytics or machine learning

## Project Outcome
This project demonstrates the ability to design and implement a data processing pipeline that ensures data quality and readiness for analysis.
